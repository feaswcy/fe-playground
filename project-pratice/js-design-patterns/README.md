## 设计模式的由来
一句话来说，设计模式是coder在长期写代码的过程中，总结的一些代码构思上比较好的方式，这些方式因为长期的积累，给他们总结出一个个名称用来指代编写的这种技巧。

## 设计模式的原则

单一职责原则、开放-封闭原则、最少知识原则、接口和面向接口编程
